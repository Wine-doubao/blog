    标签：链表 双指针 哈希表 简单

    # 题目

    给你两个单链表的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 null 。

    图示两个链表在节点 c1 开始相交：

    题目数据 保证 整个链式结构中不存在环。

    注意，函数返回结果后，链表必须保持其原始结构。

    自定义评测：评测系统的输入如下（你设计的程序 不适用 此输入）：

    intersectVal - 相交的起始节点的值。如果不存在相交节点，这一值为 0
listA - 第一个链表
listB - 第二个链表
skipA - 在 listA 中（从头节点开始）跳到交叉节点的节点数
skipB - 在 listB 中（从头节点开始）跳到交叉节点的节点数

    # 解析

    惭愧，想了一会用双指针的解法，将A中逐个节点与B中的比较，然后找到那个相交点，但是时间复杂度是O(m*n)...下面有三种方法，都是看题解看到的🤡，最没想到的是哈希集合(Set)和哈希表(Map)处处都能有身影🤠

    1. 使用哈希集合：
      • 将链表 headA 的每个节点存入哈希集合中，然后循环 headB 判断两个链表有没有相同（相交）节点，如果有则直接返回；如果没有则遍历到 null 可以直接返回。
      • 时间复杂度为O(m+n)，空间复杂度为O(m)。空间复杂度的 m 指需要使用哈希集合存储链表 headA 中的全部节点。
    2. 使用双指针（方法1）：
      • 让指针 pa 指向 headA，指针 pb 指向 headB，循环遍历
      • 如果 pa 为空，说明已经走到了链表A的尾部，pa 赋值为链表B的头结点继续遍历；pb同理
      • 如果 pa 和 pb 相遇，说明链表相交，相交的起始位置就是两个指针相遇的位置；如果 pa 和 pb 再次为空，说明链表不相交，返回null
      • 该方法利用的是指针 pa 和 pb 走的路程相同：如果链表相交且两个链表节点数相同，则两个指针一起移动肯定能相遇；如果链表相交但两个链表节点数不同，则假设 headA 节点数为 a+c，headB 节点数为 b+c，则 pa 要走 a+c+b，pb 要走 b+c+a，它们走的路程相同，最后肯定会相遇；如果链表不相交，那么 pa 和 pb 都要走 m+n，最后都为null
      • 时间复杂度为O(m+n)，空间复杂度为O(1)。时间复杂度是指两个指针同时遍历两个链表，每个指针遍历两个链表各一次。
    3. 使用双指针（方法2）：
      • 将两个链表都遍历一遍，计算出它们的节点值及相差的节点个数，然后让 pa 指向短的那个链表的头节点，pb 指向长的链表的第 n+1 个节点，这样两个指针的初始位置就相等了，再各自遍历判断当前节点是否相同，如果相同就返回当前节点，如果不相同继续遍历直到null

    还有一种方法，将其中一个链表首尾相连，然后判断另一个链表是否存在环，并找出进入环的那个节点...

    // 哈希集合
var getIntersectionNode = function(headA, headB) {
    const visited = new Set();
    let temp = headA;
    while (temp !== null) {
        visited.add(temp);
        temp = temp.next;
    }
    temp = headB;
    while (temp !== null) {
        if (visited.has(temp)) {
            return temp;
        }
        temp = temp.next;
    }
    return null;
};

// 双指针 方法1
var getIntersectionNode = function(headA, headB) {
    let pA = headA, pB = headB;
    while (pA !== pB) {
        pA = pA === null ? headB : pA.next;
        pB = pB === null ? headA : pB.next;
    }
    return pA;
};
